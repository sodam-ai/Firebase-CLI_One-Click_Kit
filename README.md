## 1. 이게 뭔가요?

- **Firebase** = 구글이 만든 **앱 만들기 도구 모음**입니다. 웹사이트를 인터넷에 올리고(호스팅), 데이터를 저장하고, 로그인 기능 등을 붙일 수 있습니다.
- **firebase-tools** = 그 Firebase를 **내 컴퓨터에서 다루는 프로그램**(명령 도구)입니다.
- **이 키트** = firebase-tools를 **초보자도 클릭만으로** 설치·사용·제거하도록 한국어로 감싼 도우미입니다.

> 한 줄 요약: **`시작하기.bat` 더블클릭 → 화면이 시키는 번호 누르기.** 그게 전부입니다.

## 2. ⚡ 빠른 시작 (3단계)

| 단계 | 무엇을 | 어떻게 |
|---|---|---|
| **1** | 키트 열기 | 폴더 안 **`시작하기.bat`** 더블클릭 |
| **2** | 설치 | 화면에서 **[1] 설치하기** 선택 (화살표 또는 번호 `1`) |
| **3** | 로그인 후 사용 | **[2] 사용하기** → 메뉴 **1) 로그인** → 이후 원하는 작업 |

> 화면 위쪽에 **"지금 이것! ←"** 로 다음에 할 일을 항상 짚어 줍니다. 그대로 따라가면 됩니다.

## 3. 사전 준비물 / 필요한 프로그램

| 준비물 | 꼭 필요? | 설명 |
|---|---|---|
| **윈도우 10 / 11** | ✅ 필수 | 이 키트는 윈도우 전용입니다. |
| **인터넷 연결** | ✅ 필수 | 설치·로그인·배포에 필요합니다. |
| **Node.js** (npm 포함) | ✅ 필수 | Firebase 도구가 돌아가는 **부품**입니다. **없어도 됩니다** — 설치 단계에서 **자동 설치를 도와줍니다.** |
| **구글 계정** | 사용 시 필요 | 로그인·프로젝트·배포에 필요합니다. (설치만 할 땐 불필요) |

> `시작하기.bat`를 켜면 화면 맨 위 **[지금 내 컴퓨터 상태]** 에서 Node.js·Firebase·로그인 여부를 자동으로 알려줍니다. 무엇이 부족한지 한눈에 보입니다.

## 4. 다운로드 방법

1. 깃허브(GitHub) 키트 페이지에서 초록색 **`< > Code`** 버튼 → **`Download ZIP`** 클릭. (또는 릴리스(Releases)의 zip 파일)
2. 받은 **zip 파일을 우클릭 → 압축 풀기**.
3. 압축을 푼 폴더 안에 `시작하기.bat`가 보이면 준비 끝.

> ⚠️ **다운로드한 파일이 막힐 때(중요):** 윈도우는 인터넷에서 받은 파일을 안전을 위해 잠가둘 때가 있습니다.
> - **가장 쉬운 방법:** 압축을 풀기 **전에** zip 파일을 **우클릭 → 속성 → 맨 아래 '차단 해제(Unblock)' 체크 → 확인**, 그다음 압축 풀기.
> - 파란 **"Windows의 PC 보호"** 창이 뜨면 → **추가 정보 → 실행**.
> - 한 번 `시작하기.bat`가 실행되면 **폴더 안 나머지 파일의 잠금은 자동으로 풀립니다.**

## 5. 설치 방법

1. **`시작하기.bat`** 더블클릭.
2. 화면에서 **[1] 설치하기** 선택 (↑↓ 화살표로 고르고 **Enter**, 또는 숫자 **`1`**).
3. 검은 설치 창이 영어로 진행됩니다. **창을 닫지 말고 1~3분 기다리세요.**
4. `ALL DONE! INSTALL WAS A SUCCESS!` 가 보이면 **설치 성공**입니다.

**Node.js(필수 부품)가 없다고 나오면?**
→ "자동으로 설치할까요? (Y/N)" 가 나옵니다. **`Y`** 입력 → 알아서 깔아줍니다.
→ 이때만 윈도우 **허용 창이 한 번** 뜰 수 있습니다(Node 설치용) → **[예]**. 안전합니다.
→ 자동 설치가 끝나면 창을 닫고 **`시작하기.bat`를 한 번 더** 실행하세요.

> 💡 설치는 관리자 권한이 **필요 없습니다.** Firebase 도구는 **내 사용자 폴더**에만 설치되기 때문입니다. (그래서 "예/아니오" 관리자 팝업이 뜨지 않습니다.)

## 6. 실행 · 사용 방법

설치가 끝나면 **`시작하기.bat` → [2] 사용하기**로 들어가면, 25가지 작업이 있는 **사용 메뉴(RUN)** 가 열립니다.

### 처음이라면 이 순서 (워크플로우)

```
[1] 로그인  →  [9] 프로젝트 만들기  →  [11] 프로젝트 선택  →  [15] 배포(인터넷에 올리기)
 (구글 연결)      (firebase init)          (사용할 프로젝트)        (전체 공개)
```

1. **로그인 (메뉴 1):** 브라우저가 열립니다 → **내 구글 계정으로 로그인** → 성공하면 창 닫기. (한 번만 하면 기억합니다.)
2. **프로젝트 만들기 (메뉴 9):** 영어로 질문이 나오면 **방향키(↑↓) + 스페이스바**로 고르고 **Enter**.
3. **프로젝트 선택 (메뉴 11):** 내 프로젝트 목록(메뉴 10)에서 본 ID를 입력.
4. **배포 (메뉴 15):** 내 앱을 **인터넷에 공개**. 안전을 위해 **`YES`(대문자)** 를 입력해야 진행됩니다.

> "배포"는 인터넷 공개라 키트가 항상 한 번 더 물어봅니다. 그냥 닫아도 아무 일 없습니다.

## 7. 작동 방식 (어떻게 동작하나요)

```
시작하기.bat ──> lib\start.ps1 (한국어 안내판: 상태 감지 + 메뉴)
                     │
                     ├─ [1] ──> INSTALL.bat    (firebase 도구 설치)
                     ├─ [2] ──> RUN.bat        (로그인/프로젝트/배포 25메뉴)
                     ├─ [3] ──> UNINSTALL.bat  (안전 제거)
                     ├─ [4] ──> 사용설명서.md  (왕초보 가이드 열기)
                     └─ [5] ──> Firebase 홈페이지 열기
```

- **`시작하기.bat`** 는 한국어 안내판(`lib\start.ps1`)을 띄우는 **유일한 문**입니다. 나머지 `.bat`는 안내판이 **자동으로 실행**하므로 직접 누르지 않아도 됩니다.
- 안내판은 내 컴퓨터 상태(Node·Firebase·로그인)를 **자동 감지**해 **다음에 할 일을 짚어** 줍니다.
- 실제 작업은 검은 창(영어)에서 진행되지만, 안내판이 한국어로 미리 설명해 줍니다.

## 8. 📂 파일 위치 / 문서 위치

### 키트 안의 파일

| 파일 | 역할 |
|---|---|
| **`시작하기.bat`** | 한국어 안내판. **여기부터 시작.** |
| `INSTALL.bat` | Firebase 도구 설치 (자동 실행됨) |
| `RUN.bat` | 로그인·프로젝트·배포 등 25메뉴 (자동 실행됨) |
| `UNINSTALL.bat` | 안전 제거 (자동 실행됨, 내 코드는 안 지움) |
| `lib\start.ps1` | 한국어 안내판의 본체 |
| `README.md` / `README.en.md` | 설명서(한/영) — 지금 이 문서 |
| `사용설명서.md` / `GUIDE.en.md` | 왕초보 가이드(한/영) |
| `LICENSE` / `NOTICE` | 라이선스 · 저작권 고지 |

### 키트가 만드는/쓰는 위치 (참고)

| 무엇 | 위치 |
|---|---|
| Firebase 도구 설치 폴더 | `C:\Users\<내이름>\AppData\Roaming\npm` (내 폴더, 관리자 불필요) |
| 로그인 정보 | `C:\Users\<내이름>\.config\configstore\firebase-tools.json` |
| 설치 기록 | 키트 폴더 안 `firebase_install.log` |
| 제거 기록 | 키트 폴더 안 `firebase_uninstall.log` |

## 9. ⌨️ 명령어 (메뉴 번호 ↔ 실제 명령)

명령어를 **외울 필요는 없습니다.** 번호만 누르면 키트가 대신 실행합니다. 아래는 "내부에서 무슨 명령이 실행되는지" 참고용입니다.

| 메뉴 | 하는 일 | 실제 실행되는 명령 |
|---|---|---|
| 1 | 로그인 | `firebase login` |
| 2 | 로그아웃 | `firebase logout` |
| 3 | 누가 로그인했나 | `firebase login:list` |
| 4 | 다시 로그인 | `firebase login --reauth` |
| 5 | 도구 업데이트 | `npm install -g firebase-tools` |
| 6 | 버전 보기 | `firebase --version` |
| 7 | Node/npm 버전 | `node -v` / `npm -v` |
| 8 | 새 버전 확인 | `npm view firebase-tools version` |
| 9 | 프로젝트 만들기 | `firebase init` |
| 10 | 내 프로젝트 목록 | `firebase projects:list` |
| 11 | 프로젝트 선택 | `firebase use <ID>` |
| 12 | 지금 쓰는 프로젝트 | `firebase use` |
| 13 | 내 PC에서 앱 실행 | `firebase serve` |
| 14 | 테스트 가짜 서버 | `firebase emulators:start` |
| 15 | 전체 배포 | `firebase deploy` |
| 16 | 호스팅만 배포 | `firebase deploy --only hosting` |
| 17 | 함수만 배포 | `firebase deploy --only functions` |
| 18 | 호스팅 사이트 목록 | `firebase hosting:sites:list` |
| 19 | 함수 로그 | `firebase functions:log` |
| 20 | 모든 명령 도움말 | `firebase --help` |
| 24 | 설치 위치 보기 | `where firebase` |
| 25 | 내 명령 직접 입력 | `firebase <내가 입력한 것>` |

> 고급 사용자는 **메뉴 25**에서 `firebase` 뒤에 붙일 부분만 직접 입력해 어떤 명령이든 실행할 수 있습니다. (예: `apps:list`)

## 10. 🛟 문제 · 오류 대처

| 이런 화면/상황 | 이렇게 하세요 |
|---|---|
| 파일이 안 열리거나 **"Windows의 PC 보호 / 차단됨"** | 받은 파일을 윈도우가 잠근 것. **우클릭 → 속성 → '차단 해제' 체크 → 확인** (위 4. 다운로드 방법 참고) |
| **`firebase ... not found`** (설치했는데 못 찾음) | 창을 **모두 닫고 새 창**에서 다시 `시작하기.bat` 실행 (윈도우가 새 프로그램을 인식하려면 새 창 필요) |
| **`Can not reach the internet`** | 와이파이/인터넷 확인 후 다시. 회사·학교 망이면 IT 담당에게 문의 |
| **`Install did not work`** | 같은 폴더 **`firebase_install.log`** 열어 마지막 줄 확인 (백신·회사망 차단이 흔함) |
| 검은 창이 영어로 떠서 무섭다 | **정상입니다.** 닫지 말고 안내판의 한국어 설명을 따라가세요 |
| 메뉴에서 엉뚱한 번호 입력 | 자동으로 메뉴로 돌아갑니다. 걱정 마세요 |

## 11. 🔐 안전 · 권한

- **설치 · 제거에 관리자 권한이 필요 없습니다.** Firebase 도구는 **내 사용자 폴더**(`AppData\Roaming\npm`)에만 설치/삭제되므로, "예/아니오" 관리자 팝업이 뜨지 않습니다.
  - 예외: **Node.js가 없어 자동 설치**할 때만 윈도우 허용 창이 한 번 뜰 수 있습니다(필수 부품 설치) → **[예]**, 안전합니다.
- **배포(Deploy)** 는 내 앱을 **인터넷에 공개**하는 작업이라, 키트가 항상 **`YES` 입력**을 한 번 더 요구합니다.
- **제거(UNINSTALL)** 해도 **내 코드 파일, Node.js, 구글 서버의 데이터는 그대로** 둡니다. firebase 도구와 그 설정/캐시만 지웁니다.

## 12. ❓ 자주 묻는 질문

**Q. 명령어를 외워야 하나요?**
A. 아니요. 번호만 누르면 됩니다. (고급 사용자는 사용 메뉴 25번에서 직접 입력 가능)

**Q. 돈이 드나요?**
A. 이 키트는 **무료**이고 요금과 무관합니다. 다만 **Firebase 서비스 자체**는 사용량이 많아지면 구글 요금(Blaze 요금제)이 생길 수 있으니, 구글 콘솔에서 확인하세요.

**Q. 잘못 눌러서 망가지면요?**
A. 위험한 작업(배포·제거)은 모두 **한 번 더 물어봅니다.** 그냥 창을 닫아도 됩니다.

**Q. 인터넷이 꼭 있어야 하나요?**
A. 설치·로그인·배포에는 필요합니다. 이미 설치된 도구의 버전 확인 등 일부는 인터넷 없이도 됩니다.
